# 管理画面 CMS 機能拡張 — 仕様書 v1.0

作成日: 2026-05-30  
対象: ren-money.com/admin  
ステータス: レビュー待ち

---

## 1. 背景と目的

現在の課題:
- X投稿はAIが自動生成するが、投稿後に文章を直したい場合に管理画面がない
- ブログ記事は自動生成されるが、画像を差し込む手段がない（WordPressのような編集UIがない）
- 記事の表現・テキストを後から修正できない

目的:
- ブログ記事を編集・画像挿入できる WYSIWYG エディタを `/admin` に追加する
- 投稿済みの X ツイートを管理・削除・修正できるパネルを追加する
- 既存の静的サイト構成（output: 'static' + GitHub Actions）を維持したまま実装する

---

## 2. 技術方針（重要）

### 既存パターンを踏襲する

```
ブラウザ（Admin UI）
  │
  ├→ 読み取り: GitHub API または ビルド時の静的データ
  │
  └→ 書き込み: GitHub API（ファイルコミット）
                 → GitHubリポジトリ更新
                 → Cloudflare Pages 自動デプロイ
```

- `output: 'static'` のまま変更しない
- SSR化・Cloudflareアダプター追加はしない
- 認証: ブラウザの localStorage に GitHub Personal Access Token を保存（既存パターン）
- 画像保存先: `/public/images/articles/` フォルダへ GitHub API 経由でコミット

### X投稿管理の書き込みについて

X API の DELETE・POST はシークレットが必要なため、ブラウザから直接呼べない。
既存の `functions/api/admin-action.ts`（Cloudflare Function）を拡張して、
X API proxy エンドポイントを追加する。
X API キーは Cloudflare Environment Variables に保存する。

## 3. 追加機能一覧

### 機能A: ブログ記事 WYSIWYG エディタ

**画面**: `/admin/edit/[slug]`

| 要素 | 内容 |
|---|---|
| エディタ本体 | TipTap（シンプルなWYSIWYGライブラリ）|
| 対応フォーマット | MDX テキスト部分の編集（フロントマターの一部を含む）|
| ツールバー | 太字 / 斜体 / 見出し(H2,H3) / リスト / リンク挿入 / 画像挿入 |
| 画像挿入 | ツールバーの「画像」→ ファイル選択 → `/public/images/articles/[slug]/` にアップロード |
| 保存 | 「保存して公開」ボタン → GitHub API でMDXファイルを更新 → 自動デプロイ |
| プレビュー | 別タブで記事URLを開く（デプロイ後に反映） |

**フロントマター編集対象**:
- `title`（タイトル）
- `description`（説明文）
- `updatedDate`（更新日、保存時に自動セット）

**フロントマター非対応**（変更しない）:
- `slug`, `pubDate`, `category` など構造的なもの

---

### 機能B: 画像アップロード

**画面**: ブログ記事エディタ内（機能Aに統合）

| 要素 | 内容 |
|---|---|
| 入力方式 | ファイル選択ダイアログ + ドラッグ&ドロップ |
| 対応形式 | JPEG / PNG / WebP（最大5MB）|
| 保存先 | `ren-blog-/public/images/articles/[slug]/[filename]` |
| 保存方法 | GitHub API `PUT /repos/.../contents/...`（base64エンコード）|
| URL形式 | `/images/articles/[slug]/[filename]` → HTMLに自動挿入 |
| トップ画像 | フロントマターの `heroImage` を書き換える専用フィールドを用意 |

---

### 機能C: X投稿 キュー＆投稿済み管理

**画面**: `/admin/x-posts`

#### C-1: 投稿キュー管理（pending）

| 要素 | 内容 |
|---|---|
| 表示内容 | 投稿予定日時 / ツイートタイプ / テキスト冒頭 / ステータス |
| 操作 | テキスト編集（インライン編集）/ 削除（status を skipped に変更）|
| 保存 | GitHub API で `tweet_queue.csv` を更新 → 自動デプロイ |

#### C-2: 投稿済みツイート管理（posted）

| 要素 | 内容 |
|---|---|
| 表示内容 | 投稿日時 / テキスト / tweet_id / Xへのリンク |
| 操作 | **削除**: Cloudflare Function → X API DELETE |
| 修正方法 | テキスト編集 → 「修正」押下 → 旧ツイート削除 → 新規ツイート投稿 |
| 注意表記 | 「修正はツイートの削除→再投稿になります（X API仕様）」と明示 |

## 5. API 設計

### 既存の Cloudflare Function を拡張

**ファイル**: `functions/api/admin-action.ts`（既存）に追記

```
POST /api/admin-action
  action: "github_commit"   → 既存（記事保存・画像アップロード）
  action: "x_delete_tweet"  → 新規（X API DELETE）
  action: "x_post_tweet"    → 新規（X API POST）
```

### 必要な Cloudflare Environment Variables（追加）

| 変数名 | 用途 |
|---|---|
| `X_API_KEY` | X API キー |
| `X_API_SECRET` | X API シークレット |
| `X_ACCESS_TOKEN` | X アクセストークン |
| `X_ACCESS_TOKEN_SECRET` | X アクセストークンシークレット |
| `GITHUB_TOKEN` | GitHub API トークン（既存） |
| `GITHUB_REPO` | `Lofikota/ren-blog-`（既存） |
